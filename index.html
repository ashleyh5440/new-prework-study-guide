Git Bash/Command Line

  • pwd current working directory
  • ls list of files/folders
  • cd takes you to home directory
  • cd [destination] move between directories
  • cd .. navigate backwards in directory
  • mkdir [directory name] creates new directory
  • rm -rf [directory name] deletes directory
  • touch [file name] creates file in directory
  • rm [file name] deletes file in directory
  • start opens a file
  • git status shows current branch, checks if you're in a repo
  • git checkout -b [branch name] creates new branch
  • git clone [paste SSH key or HTTPS]
  • clones repo to machine
  • git pull origin main syncs local branch with main branch
  • git push origin [branch name] pushes local changes to GitHub branch
  • git commit -m "[comments]" adds commits to the repo
  • To push local changes to GitHub, do the following steps:
    1. git add -A
    2. git commit -m "[comments]"
    3. git push origin [branch name]

GitHub

  • To create a new repository in GitHub:
    1. Select New repository and type in the name
    2. Select Public or Private
    3. Select Add a README file
    4. Click Create repository
  • To clone a repository:
    1. Navigate to the main page of repository
    2. Click <>Code
    3. Copy URL using SSH key
    4. In Bash, put in git clone [paste SSH link]

  • <pre> preserves spaces spaces and line breaks

CSS

Three ways to style CSS: inline CSS, internal CSS stylesheet, external CSS stylesheet


  • inline CSS: style directly in HTML file, apply to specific tag
    • 
        <h1 style="color: red"> Rework Study Guide </h1>
  • internal stylesheet: embed entire stylesheet directly in HTML file, applies to whole page
  • external stylesheet: most common, keeps CSS rules in separate file

CSS rule example


  • div {color: blue; text-align: justify}
  • div: selector, defines the element/attribute
  • color:blue; , text-align:justify: declaration, contains the property (color/text-align) and value (blue/justify)

Formatting/display


  • , combines two selectors into a single CSS rule so declarations affect both
  • .c period before the c designates it as a class selector
  • content made up of the height and width of content
  • padding indicates how much space is around the content inside an element
  • border extends content and padding to include border
    • border has three components that need to be added: size (__px), style (solid, dashed, doubled, ect.), and color
  • margin indicates how much space is around the outside of an element

JavaScript

  • variables: containers that store values. Start with let keyword followed by variable name
    • 
            let myVariable;
  • comments text that can be added to code, ignored by browser
    • 
         /*
           Everything in between is a comment
         */
  • operators: mathematical symbols that produce results based on two values/variables
  • conditionals: code structures used to test if an expression returns true. Commonly seen as if . . . else statements
    • 
            let iceCream = "chocolate";
            if (iceCream === "choclate") { 
              alert ("Yay, I love chocolate ice cream!");
            } else {
               alert ("Awwww, but chocolate is my favorite . . .");
                }
  • function: way of packaging functionality to reuse. Often take on aguments
    • let myVariable = document.querySelector (h1);
  • argument: value passed as input a function
  • event handlers: code structures hat listen for activity in the browser and run code in response. Example is the click event, when you click something with the mouse
  • var topics = "HTML, CSS, Git, JavaScript";
    • var: declares a variable
    • topics: name of the variable/container
    • ". . .": data stored by the variable/container, called a string
  • console.log: method to output a message to web console by adding an argument
    • console: test environment developers use to check code
    • .log: method that outputs content in parentheses to the console
  • is statement: code will only execute if the statement is true/meets a condition
    • if (condition) {
      // block of code to be executed
      }
  • array: variable that holds a group of data, usually data that's related
    • Ex: var shapes = ["triangle", "square", "pentagon", "circle"];
  • index: identifies the location/order of data in an array. Starts at 0
  • for loop: uses predictable pattern to perform a task on all items in array by allowing a single block ofcode to un over and over
    • 3 statements in a for loop:
      • starting point: var x=0 starts loop at beginning of array
      • condition: x<shapes.length; length of array, which determines how long loop will run
      • final: what allows the array to iterate over each item. x++ makes sure every time the code loops, it increses by one
    • var shapes =["triangle", "square", "pentagon", "circle"];
      for (var x=0; <shapes.length; x++){
      console.log(shapes[x]);
      }
  • function: set of instructions in JavaScript that tells computer how to act
    • 
            function helloWorld () {
            // code block
            }
